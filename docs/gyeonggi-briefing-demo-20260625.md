# CowTalk 경기도 인수위 브리핑 — 시연 대본

> 대상: 경기도 인수위원회 (경기도 축산 행정 대전환)
> 장소: 우리 목장 (현장 방문)
> 목적: "경기도가 왜 이 사업을 해야 하는가" = **기대효과**를 시연으로 증명
> 총 소요시간: 약 15분 (브리핑 슬라이드 별도)

---

## 0. 핵심 원칙 — 기능이 아니라 "기대효과"를 보여준다

모든 클릭은 **"경기도가 얻는 것"**으로 번역해서 말한다.
화면 자랑이 아니라, "이걸로 경기도 축산 행정이 어떻게 바뀌나"를 보여주는 것이 목표.

> **한 줄 메시지 (처음과 끝에 반복):**
> "경기도를 대한민국 축산 디지털 행정의 첫 모델로 만드는 운영체제입니다.
> 센서가 아니라, 알람을 '행동'과 '행정'으로 바꾸는 다리입니다."

## 1. 시연 전 체크리스트 (시연 15분 전 + 직전)

### 안정성 — 가장 큰 변수는 코드가 아니라 콜드스타트
- [ ] **서버 pre-warm** (콜드스타트 15~20초 방지): `curl -s https://<운영서버>/api/health`
- [ ] 시연용 계정으로 미리 로그인 1회 → 서버·캐시 warm-up
- [ ] **각 섹션 직전** warm-up 질문 1회 (프롬프트 캐시 TTL 5분 — 5분 이상 쉬면 다시 느려짐)
- [ ] Claude API 키 잔량 확인
- [ ] 네트워크: 현장 WiFi/LTE 사전 테스트
- [ ] **사전 녹화 백업** — 아래 5단계 시퀀스를 미리 녹화 (콜드스타트·네트워크 사고 대비)
- [ ] 주요 화면 스크린샷 백업

### 정확도 — "데이터 없음"이 뜨지 않게
팅커벨은 정직해서 데이터가 없으면 "데이터가 없습니다"라고 답한다(데모에선 약해 보임).
- [ ] **"경기도 발열 현황 알려줘"** 미리 1회 질문 → **실제 숫자가 나오는지** 확인
- [ ] "데이터 없음"이 뜨면 → 경기 방역/발열 시드 보강 후 재확인
- [ ] 시연 개체(예: 423번)·시연 농장(포천 등)에 이벤트가 실제로 떠 있는지 확인

### 개인정보
- [ ] 실제 농가·개체 번호 노출 주의 → 데모 계정 사용 또는 "실데이터입니다" 사전 양해

## 3. 메인 시연 — 5분 임팩트 시퀀스

### 🎬 도입 멘트
> "경기도 포천에서 구제역 의심 1건이 발생했다고 가정해 보겠습니다.
> 지금 경기도 방역 행정으로는 어떻게 대응하고, CowTalk으로는 어떻게 달라지는지 보시겠습니다."

방역관(quarantine@test.kr)으로 로그인 → `/epidemiology/dashboard`

| 단계 | 화면 / 행동 | 말할 기대효과 (숫자로) |
|---|---|---|
| **① 전국·경기 현황** | 팅커벨에 **"경기도 발열 현황 알려줘"** 입력 | "예전엔 시군에 전화 돌려 며칠. 지금은 **3초**에 경기 52개 농장 발열 현황이 한 화면에." |
| **② 위험농장 드릴다운** | 위험농장 TOP5 → 포천 농장 클릭 → `/farm/:id` | "증상이 나오기 **전에** 위내센서가 체온·활동 이상을 감지. 확산 **전에** 차단." |
| **③ 접촉망 추적** | `/epidemiology/contact-network` | "이 농장과 소가 오간 모든 농장을 **30초**에 식별. 손으로 하던 역학조사가 즉시." |
| **④ 확산 시뮬레이션** | `/epidemiology/simulation` | "확산 규모·살처분 범위를 **사전 예측**. 구제역 1회 발생 = **수십~수백억** 보상·살처분 비용. 한 번만 막아도 시스템값을 회수." |
| **⑤ 행정 관점 전환** | admin@gyeonggi.kr로 전환 → `/dashboard` | "같은 데이터를 행정관은 **수급·예산 관점**으로 봅니다. 아날로그 축산 행정의 디지털 전환." |

> **닫는 멘트:**
> "경기도는 이미 우리 네트워크 146개 농장 중 **52곳으로 1위**입니다.
> 데이터가 가장 많은 곳이 표준을 만듭니다. 경기도가 대한민국 축산 디지털 행정의 첫 모델이 됩니다."

## 4. 팅커벨(AI 비서) 시연 — 왜 강력한가

경기 시연의 핵심 무기. 시연 시 **속도와 정확도**를 동시에 보여준다.

### 추천 질문 (방역 모드 — 방역관 로그인 시 자동 활성화, 오렌지 테마)
1. "경기도 발열 현황 알려줘" → `query_national_situation` (시도별 데이터)
2. "위험 농장 TOP5 보여줘" → `query_quarantine_dashboard`
3. "포천 지역 상황 어때?" → 포천 → 경기도 자동 매핑 + 시도 상세

### 시연 시 강조 포인트
- **속도**: 스트리밍으로 첫 토큰 1~2초, Sonnet 4.6 + 프롬프트 캐싱 + 도구 병렬 실행.
  → "AI가 스스로 필요한 도구를 골라 DB를 직접 조회합니다."
- **정확도**: 모든 수치는 **DB 조회값만** 사용(환각 방지 가드 + 출처 기록 의무화).
  → "팅커벨은 추측하지 않습니다. 답변의 모든 숫자에 출처가 붙습니다."
- **지역 인식**: "경기 / 경기도 / 포천" 모두 경기도로 정확 매핑.

## 5. 경기도 기대효과 — 4줄 요약 (슬라이드/구두용)

- 💰 **예산 절감** — 전염병 1회 차단 = 수십~수백억 / 조기감지로 폐사·손실 최소화
- ⚡ **행정 속도** — 며칠 걸리던 역학조사가 30초, 엑셀·전화가 대화 한 줄로
- 🛡️ **방역 주권** — 146농장 실시간 통합 모니터링 + 확산 사전 시뮬레이션
- 🌾 **농가 소득** — 번식성적 향상·질병 조기대응으로 도내 농가 수익 증가

## 6. 경기도에 드리는 요청 (마무리)

- **시범사업(파일럿)**: 경기도 일부 시군 + 공수의사 대상 실증
- **데이터 연계 지원**: KAHIS·이력제·도 방역 데이터 연동 행정 협조
- **확산 로드맵 공동 수립**: 경기도 전역 → 전국 표준화
