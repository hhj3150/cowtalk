# 팅커벨 음성 대화 현장 검증 체크리스트

**적용 버전**: 2026-05-13 이후 (커밋 199e495 이상)
**대상 환경**: 해돋이목장(674) 또는 동등 환경 — 실제 축사 소음 포함
**필요 장비**: 안드로이드 휴대폰 1대 + 아이폰 1대 + (선택) AirPods/이어폰

## 사전 점검

- [ ] `https://cowtalk.netlify.app` 접속, 농장주 또는 시연 계정으로 로그인
- [ ] 팅커벨 FAB(우하단 보라색 요정 아이콘)가 표시되는가
- [ ] 휴대폰 마이크 권한 — 첫 호출 시 허용 (이후 영구 허용)

## 1. Push-to-Talk (PTT) — 짧은 탭 vs 길게 누르기

### 1.1 짧은 탭 (기존 토글)
- [ ] 마이크 버튼을 짧게 탭(< 250ms) → 토글로 마이크 ON
- [ ] 다시 탭 → 마이크 OFF + 전사 시작
- [ ] 응답이 음성으로 자동 재생되는가

### 1.2 길게 누르기 (PTT 신규)
- [ ] 마이크 버튼을 250ms 이상 누르고 있기 → 펄스 강화 + scale 1.08
- [ ] 누르고 있는 동안 말하기
- [ ] 손을 떼면 즉시 전사 → AI 응답
- [ ] **장갑 끼고 시도** → 작동 여부 확인 (목장주 시나리오)

### 1.3 안전성
- [ ] 누른 채로 화면을 떠나거나 다른 앱 전환 → 마이크가 영구 켜져있지 않은가
- [ ] 멀티터치(두 손가락 동시 터치) → 두 번째 터치 무시되는가

## 2. VAD — 침묵 자동 종료 (iOS Whisper 경로)

### 2.1 정상 발화
- [ ] 마이크 ON → "이 소 언제 수정하면 좋아?" 같은 짧은 발화
- [ ] 말이 끝나고 1초 후 자동으로 전사 시작 (사용자가 정지 버튼 안 누름)
- [ ] 입력 바에 파형이 움직이는가 (volume 시각화)

### 2.2 보정 단계
- [ ] 마이크 ON 직후 0.5초간 "환경 조정 중..." 표시
- [ ] 환경 조정 후 "말씀하세요" → 발화 시작 → "듣는 중..." 전환

### 2.3 축사 소음 환경 (가장 중요)
- [ ] 환기팬 소리 있는 곳에서 마이크 ON → 1초 후 침묵 오탐 없는가
- [ ] 멀리서 소가 우는 소리 → 발화로 오인식 안 되는가
- [ ] 적응형 baseline이 noise floor를 흡수하는지 (`thresholdMultiplier=2.5`)

### 2.4 안전망
- [ ] 8초 이상 계속 말하면 자동 정지되는가 (VAD 실패 시 보호)

## 3. TTS↔STT 피드백 루프 차단

### 3.1 AI 답변 중 자기 음성 차단
- [ ] AI가 음성으로 답변하는 동안, 마이크가 자기 음성을 듣지 않는가
- [ ] wake word 활성화 상태에서도 AI가 "팅커벨이..." 같은 문구를 말해도 wake 발동 안 되는가
- [ ] AI 답변 종료 후 200ms 뒤에 wake word 재개되는가

### 3.2 Barge-in
- [ ] AI 답변 중에 마이크 버튼 탭 → 즉시 답변 중단 + 듣기 시작
- [ ] "조용히 해" 발화 → 답변 중단

### 3.3 연속 대화
- [ ] 첫 질문 → 답변 → 두 번째 질문 → 답변 — 자연스럽게 이어지는가
- [ ] 답변 중 마이크가 발화로 잘못 인식해 새 질문이 들어가지 않는가

## 4. 모바일 바닥시트 + FAB

### 4.1 FAB 기본 상태
- [ ] 화면 우하단(MobileBottomNav 위)에 FAB 표시
- [ ] AI가 생각·말하는 중에는 펄스 애니메이션
- [ ] AI 응답 도착 시 미읽음 배지(빨간 원 + 숫자) 표시

### 4.2 시트 열기
- [ ] FAB 탭 → 바닥시트가 아래에서 슬라이드 업
- [ ] "팅커벨" 호명 → 시트 자동 열림 (안드로이드만)
- [ ] 시트 열림 시 미읽음 배지 즉시 0으로

### 4.3 시트 닫기
- [ ] 상단 드래그 핸들(회색 막대) 아래로 60px 이상 스와이프 → 닫힘
- [ ] 드래그 핸들 짧게 드래그(< 60px) → 원위치 (실수 방지)
- [ ] 백드롭(어두운 영역) 탭 → 닫힘
- [ ] ✕ 버튼 → 닫힘

### 4.4 한 손 조작
- [ ] 마이크 버튼이 엄지로 닿는가
- [ ] FAB가 엄지로 닿는가

## 5. TTS 쿼터

### 5.1 잔여 한도 확인
- [ ] `GET /api/audio/usage` 호출 → 본인 일/월 사용량 + 잔여 한도 반환
- [ ] government_admin/quarantine_officer는 `bypass: true`

### 5.2 한도 초과 동작
- [ ] (수동 트리거) Redis에 일 한도(50,000자) 강제 채워서 한 번 더 호출 → 429 응답
- [ ] 응답 헤더 `Retry-After: <초>`, body `error.code: TTS_QUOTA_EXCEEDED`
- [ ] 클라이언트에 "오늘 음성 사용량 한도에 도달했습니다" 메시지 표시

## 6. 다국어 코드스위칭

### 6.1 한국어 + 영어 브랜드 혼합
- [ ] "smaXtec 데이터 좀 보여줘" → 'smaXtec'이 한글로 변환되지 않고 라틴 표기로 유지
- [ ] "CowTalk에서 어제 발정 알람" → 'CowTalk' 라틴 유지
- [ ] "DIM 200일 넘은 소" → 'DIM' 라틴 유지

### 6.2 언어 전환 (LangSwitcher)
- [ ] 한국어 → 영어 전환 후 마이크 → 영어로 인식
- [ ] 영어 → 우즈벡어 전환 후 마이크 → 우즈벡어 인식

## 7. 파형 + 단계별 상태

- [ ] 듣는 중 → 입력 바에 빨간 파형 + "듣는 중..." 텍스트
- [ ] 생각 중 → 입력 바에 주황 펄스 도트 + 전사 미리보기 `"[발화 내용]" 전사 중...`
- [ ] 응답 중 → AI 답변이 메시지로 나타남

## 8. 알려진 한계 + 회피

### iOS Safari 한계
- [ ] iOS는 wake word 비활성화됨 (`platformLimitation === 'ios'`) — 정상 동작
- [ ] iOS는 MediaRecorder + Whisper 경로 사용 (안드로이드보다 1~3초 느림)

### 네트워크 약한 환경
- [ ] 4G 신호 약한 곳에서 발화 → Whisper 응답 5~10초 — 사용자에게 "생각 중..." 보이는가
- [ ] 응답 안 와도 8초 안전망에서 마이크 OFF

다음 중 하나라도 발생하면 직전 안정 커밋(67352df 또는 그 이전)으로 롤백:
1. 마이크가 영구히 켜져있는 사고 (배터리/프라이버시 위험)
