## 2. 알람별 알고리즘 역분석

### 2.1 체온 이상 (Temperature)

#### 2.1.1 temperature_high (고열)

- **이벤트 수**: 77,139건 (전체 2위)
- **발생일 체온**: 평균 39.35°C, P50=39.28, P75=40.07, P90=40.78
- **전후 변화**: 체온 **+0.58°C** 상승 (정상 대비 +1.9%)
- **동반 변화**: 반추 -20.5분/일 (-4.1%), 활동 미변화

**추정 알고리즘:**
```
IF 일평균 체온 > 39.4°C (정상 P95 = 39.60 기준, 약간 보수적)
   AND 이전 3일 대비 +0.4°C 이상 상승
THEN temperature_high 발동
심각도: > 40.0°C → critical, > 39.7°C → warning, > 39.4°C → caution
```

#### 2.1.2 temperature_low (저체온)

- **이벤트 수**: 31,222건
- **발생일 체온**: 평균 38.18°C, P25=38.28, P10=37.68
- **전후 변화**: 체온 **-0.24°C** 하강
- **동반 변화**: 반추 -25분/일, 활동 소폭 감소

**추정 알고리즘:**
```
IF 일평균 체온 < 37.5°C (정상 P5 = 37.25 근처)
   OR 이전 3일 대비 -0.3°C 이상 하강
THEN temperature_low 발동
주의: 분만 직전 체온 하강(정상)과 구분 필요 → 임신 상태 체크
```

### 2.2 반추 이상 (Rumination)

#### 2.2.1 rumination_decrease (반추 감소)

- **이벤트 수**: 108,728건 (전체 1위, 가장 빈번)
- **발생일 반추**: 평균 371.5분, P50=382.0 (정상 P50=524.6 대비 27% 감소)
- **전후 변화**: 반추 **-78.2분/일** (-17.5%), 가장 큰 변화
- **동반 변화**: 체온 미변화, 활동 미변화

**추정 알고리즘:**
```
최근3일_반추평균 = avg(최근 3일 반추)
이전4일_반추평균 = avg(이전 4~7일 반추)
감소율 = (이전 - 최근) / 이전

IF 감소율 > 15%  (절대값 약 80분/일 이상)
   AND 최근3일_반추평균 < 420분/일  (정상 P25 이하)
THEN rumination_decrease 발동
심각도: 감소율 > 30% → warning, > 20% → caution
```

> 이 알람은 가장 빈번하며, 케토시스/산독증/유방염/사료 변경 등 다양한 원인의 비특이적 지표.
> 단독으로는 진단 불가. 체온+활동과 조합하여 감별진단 필요.

### 2.3 활동량 이상 (Activity)

#### 2.3.1 activity_increase (활동 증가)

- **이벤트 수**: 49,829건
- **발생일 활동**: 평균 4.35, P75=5.62, P90=7.73 (정상 평균 2.84의 1.5배)
- **전후 변화**: 활동 **+0.78** (+44.1%), 반추 -38.2분 (-8.1%)
- **동반 변화**: 체온 +0.28°C (미미)

**추정 알고리즘:**
```
IF 당일 활동 > 개체별_7일_평균 × 1.5 (150%)
   OR 당일 활동 > 5.0 (정상 P95 = 5.58 근처)
THEN activity_increase 발동
주의: 발정(estrus)과 유사 패턴 → 반추 감소+체온 변화 없으면 발정 가능성
```

#### 2.3.2 activity_decrease (활동 감소)

- **이벤트 수**: 9,813건
- **발생일 활동**: 평균 2.84, P25=1.22
- **전후 변화**: 활동 **-2.74** (-43.5%), 반추 -64.6분 (-12.7%)
- **동반 변화**: 체온 -0.17°C

**추정 알고리즘:**
```
IF 당일 활동 < 개체별_7일_평균 × 0.5 (50% 이하)
   AND 3일 연속 감소 추세
THEN activity_decrease 발동
주의: 제엽염(파행)의 핵심 지표. 반추 동반 감소 시 통증/전신질환 의심.
```

### 2.4 발정 감지 (Estrus)

#### 2.4.1 estrus (발정)

- **이벤트 수**: 75,850건 (전체 3위)
- **센서 패턴 (핵심)**:
  - 활동: +0.92 (+60.4%) — **가장 특징적 신호**
  - 반추: -67.0분 (-13.6%) — 발정 행동으로 반추 시간 감소
  - 체온: +0.04°C (거의 무변화, 일부 +0.3~0.5°C)
- **발정일 센서**: 활동 P50=3.39 (발정 전 약 2.5), 반추 P50=445 (발정 전 약 510)

**추정 알고리즘:**
```
활동변화 = (당일 활동 - 7일 평균) / 7일 평균
반추변화 = (7일 평균 반추 - 당일 반추) / 7일 평균

IF 활동변화 > 40%  (평소의 1.4배 이상)
   AND 반추변화 > 10%  (반추 10% 이상 감소 동반)
   AND 체온 변화 < +1.0°C  (열성 질환 배제)
THEN estrus 발동

정밀 조건:
  - 직전 발정으로부터 18~24일 경과 시 가산점 (21일 주기)
  - DIM > 20일 (분만 후 자궁 회복기 제외)
  - 건유우/임신 확인 개체 제외
```

#### 2.4.2 estrus_dnb (Did Not Breed — 발정 미수정)

- **이벤트 수**: 166건
- **발정 감지 후 수정이 이루어지지 않은 사례 자동 기록**

**추정 알고리즘:**
```
IF estrus 발생 후 48시간 이내 insemination 이벤트 없음
THEN estrus_dnb 발동
용도: 놓친 수정 기회 추적, 번식 효율 분석
```

### 2.5 건강 이상 (Health)

#### 2.5.1 health_general (종합 건강 이상)

- **이벤트 수**: 53,605건
- **센서 패턴**:
  - 체온: 38.56°C (정상 수준이나 분산 큼, P90=40.66)
  - 반추: -30.5분 (-8.2%)
  - 활동: -0.04 (거의 무변화)

**추정 알고리즘:**
```
IF (반추 감소 > 8% AND 체온 정상범위 초과)
   OR (반추 감소 > 15% AND 활동 감소 > 10%)
   OR (체온 > 39.5°C AND 반추 감소 동반)
THEN health_general 발동
특징: 비특이적 건강 이상 신호. 특정 질병으로 분류되지 않는 복합 이상.
```

#### 2.5.2 clinical_condition (임상 상태 이상)

- **이벤트 수**: 10,207건
- **센서 패턴**:
  - 체온: 38.25°C (분산 매우 큼, P90=40.09, min=6.61 — 센서 이탈 포함)
  - 반추: -34.4분 (-7.0%)
  - 활동: -0.16

**추정 알고리즘:**
```
IF health_general 조건 충족
   AND (체온 > 40.0°C OR 반추 감소 > 20% OR 활동 감소 > 30%)
THEN clinical_condition 발동 (health_general보다 심각한 상위 알람)
주의: 센서 이탈(체온 극저값)도 이 이벤트에 포함됨 → 센서 상태 체크 필요
```

### 2.6 분만 감지 (Calving)

#### 2.6.1 calving_detection (분만 감지)

- **이벤트 수**: 17,702건
- **센서 패턴 (핵심)**:
  - 체온: **-0.32°C 하강** (분만 전 24~48시간)
  - 반추: -34.2분 (-8.2%)
  - 활동: +0.30 (미미한 증가, 일부 급증)
- **발생일 체온**: P50=38.70°C (정상보다 약간 낮음)

**추정 알고리즘:**
```
전제: 건유우 OR 임신 후기 (예상 분만일 ± 15일)

IF 24시간 내 체온 -0.3°C 이상 하강 (개체별 7일 평균 대비)
   AND 반추 감소 동반 (>5%)
   AND 활동 불규칙 (진통으로 인한 패턴 변화)
THEN calving_detection 발동

시간 해상도가 중요:
  - 6시간 단위 체온 추적 → 하강 시점 포착
  - 분만 전 6~12시간: 체온 최저점 + 활동 급증 (산통)
```

#### 2.6.2 calving_waiting (분만 대기)

- **이벤트 수**: 570건
- 예상 분만일 접근 시 자동 생성 (센서 패턴 아닌 달력 기반)

#### 2.6.3 calving_confirmation (분만 확인)

- **이벤트 수**: 16,461건

## 4. 자체 센서 개발 시 핵심 요구사항

### 4.1 센서 하드웨어

| 지표 | 필수 해상도 | 수집 주기 | 정밀도 |
|------|-----------|-----------|--------|
| 체온 | 0.01°C | 10분 이하 | ±0.1°C |
| 반추 | 1분 단위 | 10분 집계 | ±5분 |
| 활동 | 3축 가속도 → index | 10분 집계 | ±10% |

### 4.2 알고리즘 우선순위

| 순위 | 알람 | 구현 난이도 | 임상 가치 | 비고 |
|------|------|-----------|-----------|------|
| 1 | estrus (발정) | 중 | **최고** | 수태율 직결, 경제 효과 최대 |
| 2 | temperature_high | 하 | 높음 | 단순 임계값, 질병 조기 발견 |
| 3 | rumination_decrease | 하 | 높음 | 비특이적이지만 가장 빈번 |
| 4 | calving_detection | 상 | 높음 | 체온 하강 + 시간 해상도 필요 |
| 5 | health_general | 중 | 중 | 복합 조건, 감별진단 필요 |
| 6 | activity_decrease | 하 | 중 | 파행/통증 지표 |
| 7 | clinical_condition | 중 | 높음 | 중증 지표, 수의사 필수 |

### 4.3 smaXtec 대비 차별화 기회

1. **AI 감별진단**: smaXtec는 알람만 줌. 우리는 "왜 이 알람이 떴는가" + "뭘 해야 하는가"까지
2. **패턴 학습**: 개체별 베이스라인 자동 보정 (개체마다 정상 체온이 다름)
3. **복합 알람**: 3개 지표 동시 분석으로 감별진단 정확도 향상
4. **시간 해상도**: 10분 데이터를 6시간 윈도우로 분석하면 분만 감지 정확도 향상
5. **공공데이터 연동**: 혈통+DHI+이력제 → 개체별 위험도 사전 보정
