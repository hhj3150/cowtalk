## 1. 행정관 대시보드 (5분) — 전체 현황 파악

**로그인**: ha@d2o.kr (Master Admin)
**화면**: `/dashboard`

### 1-1. AI 일일 브리핑
- Claude AI가 자동 생성한 오늘의 요약
- 알림 건수, 전일 대비, 심각/높음 카운트
- "AI가 매일 아침 146개 농장을 한 문장으로 요약합니다"

### 1-2. 농장/그룹 선택
- 그룹(5개) → 146개 농장 드롭다운
- 특정 농장 선택 시 대시보드가 해당 농장으로 필터링

### 1-3. KPI 카드
- 총 두수, 센서 장착, 건강 알림, 발정 감지 등
- 각 카드 클릭 → 드릴다운 상세 (시연 시 1~2개만)

## 2. 개체 디지털 트윈 (5분) — AI가 소를 읽는다

**화면**: `/cow/{id}` (423번 등 이벤트 풍부한 개체)

### 2-1. 개체 프로필
- 이력번호, 품종, 산차, 번식상태
- smaXtec 센서 데이터 + 축산물이력추적시스템(data.ekape.or.kr) 공공데이터 연동

### 2-2. 30일 센서 차트
- 체온/반추/활동 트렌드
- AI 건강 점수 (50~100점 범위)

### 2-3. 소버린 AI 버튼
- 클릭 → Claude AI가 실시간으로 개체 상태 해석
- "이 소의 현재 상태는... 추천 조치는..."
- **핵심 메시지**: "한국 데이터는 한국 AI가 해석합니다 (소버린 AI)"

### 2-4. 이벤트 타임라인
- 발정/수정/임신감정/질병 이벤트 히스토리
- "센서 알람 → AI 판단 → 행동 추천 → 기록"이 한 화면에서 완결

## 3. 방역관 대시보드 (5분) — 146농장 통합 방역

**역할 전환**: 방역관(quarantine_officer)
**화면**: `/epidemiology/dashboard`

### 3-1. 방역 KPI
- 위험 농장 수, 24시간 발열 트렌드, 미처리 조사 건수
- "146개 농장의 방역 상황을 한 화면으로"

### 3-2. 지역 인텔리전스 지도
**화면**: `/regional-map`
- 시도 경계선 + 비례 마커 (두수 비례)
- 위험도별 색상 (정상/주의/위험/심각)
- 필터: 농장상태 / 발정활동 / 건강위험 / 센서활성도
- "전국 축산 현황을 지도 한 장으로"

### 3-3. 접촉 네트워크
**화면**: `/epidemiology/contact-network`
- 농장 간 이동 이력 기반 접촉 그래프
- 위험 전파 경로 시각화
- "구제역 발생 시 30초 내 전파 경로 파악"

### 3-4. 확산 시뮬레이션 (시간 여유 시)
**화면**: `/epidemiology/simulation`
- SEIR 모델 기반 확산 예측
- "발생 시 피해 규모를 사전 시뮬레이션"

## 6. 클로징 (2분)

### 핵심 메시지 3가지:

1. **소버린 AI**: 한국 축산 데이터는 한국 AI가 해석 — 데이터 주권 확보
2. **센서→행동 완결**: smaXtec 알람 → AI 판단 → 역할별 액션플랜 → 기록이 한 플랫폼에서 완결
3. **확장 가능**: 1농장 → 146농장 → 지역 → 국가 → 해외 수출 가능 구조

### 중앙아시아 확장 메시지:
- "한국에서 검증된 플랫폼을 국가별 어댑터로 현지화"
- "smaXtec 센서 + CowTalk AI = 글로벌 축산 디지털 전환 패키지"
