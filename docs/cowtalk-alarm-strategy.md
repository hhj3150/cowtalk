## 핵심 철학

smaXtec의 한계: "알람만 줌. 그래서 뭘 하라는 건지 모름."
CowTalk의 차별점: **알람 → 판단 → 추천 → 행동 → 기록 → 학습**이 한 화면에서 끝남.

## 1. 발정 발견 + 수정 코치

### 1.1 현장의 문제

소는 주로 야간에 발정이 오고, 사람이 관찰하지 않는 시간에 발생함.
발정을 놓치면 21일을 더 기다려야 하고, 1회 놓칠 때마다 **두당 30만원 이상** 손실.

### 1.2 요구 기능 (5단계 완결형)

| 단계 | 기능 | 현재 상태 |
|------|------|-----------|
| ① 발정 발견 | 센서 데이터 기반 자동 감지 (사람 개입 없이) | ✅ estrus 룰 구현 (활동+60%, 반추-14%) |
| ② 수정 적기 | 발정 시작 시각 기준 최적 수정 시간대 계산 | ✅ AM-PM Rule + 목장별 윈도우 |
| ③ 수정 의견 | 이 소에게 지금 수정이 적절한가? | ✅ 부분 구현 (육성우 나이, DIM, 건강, 반복수정) |
| ④ 정액 추천 | 최적 정액 선택 (3단계 필터) | ⚠️ 목장 보유 정액만 (전국 카탈로그 미구현) |
| ⑤ 기록+학습 | 수정 결과 → 수태율 피드백 → 다음 추천 개선 | ✅ 수태율 학습 루프 구현 |

### 1.3 수정 의견의 세부 판단 기준

현재 구현된 것:
- 육성우: 월령 < 목장 기준(보통 12~14개월) → "아직 이름"
- 경산우 DIM < 50일 → "분만 후 자궁 회복기, 다음 발정 대기"
- 최근 30일 건강 이벤트 → "건강 상태 확인 후 수정 권장"
- 연속 미임신 3회+ → "번식장애 검진 필요"
- 직전 수정 후 15~24일 → "발정재귀(수태 실패 가능)"
- DIM > 200일 → "장기공태우 — 번식장애 검진"

**추가 필요:**
- 분만 후 자궁 회복 상태 추정 (센서 기반 — 체온 안정화 패턴)
- BCS(체충실지수) 센서 추정 (활동량+반추 패턴으로 간접 추정)
- 수정란 이식 대상 판별: 발정 후 7일 뒤 이식 적기 알림

### 1.4 정액 추천 3단계

```
[1단계] 목장 보유 정액 중 최우선 추천
  ├─ 동일 품종 필터 (한우↔젖소 교차 금지)
  ├─ 근교계수 체크 (동일 종모우 재사용 경고)
  ├─ 목장 내 과거 수태율 기반 학습 가산점
  └─ 유전능력(유량, 체형 등) + 재고 + 가격 종합 점수

[2단계] 전국 유통 정액 중 추천 (목장 재고 부족 시)
  ├─ 국산 정액: 한우(KPN) + 젖소(KH) 카탈로그
  ├─ 수입 정액: ABS, Semex, CRI, Alta 등
  ├─ 성감별 정액 옵션
  └─ 축주가 추구하는 방향 반영 (유량 중시 vs 체형 vs 장수성)

[3단계] 지구상 최적 정액 (컨설팅 모드)
  ├─ CDCB (미국), Interbull, CRV (네덜란드) 데이터 연동
  ├─ 유전체 분석 기반 교배 계획
  └─ 근교 최소화 + 유전능력 최대화 조합 최적화
```

**현재 1단계 구현 완료. 2~3단계는 정액 카탈로그 확장 필요.**

### 1.5 알고리즘 (실측 데이터 기반)

```
발정 감지 조건 (실측 53만 건 분석):
  활동: 개체 7일 평균 대비 +40% 이상 (실측 평균 +60.4%)
  반추: 개체 7일 평균 대비 -10% 이상 감소 (실측 평균 -13.6%)
  체온: +1.0°C 미만 (열성 질환 배제)
  제외: DIM < 20일, 건유우, 임신 확인 개체, 직전 발정 14일 이내

수정 적기 (AM-PM Rule):
  오전 발정 → 당일 오후 수정 (적기 10~18h)
  오후 발정 → 다음날 오전 수정 (적기 12~20h)
  목장별 설정값 적용 (해돋이목장: 발정 후 10~18h)
```

## 2. 임신 유무 — 데이터 기반 혁신

### 2.1 현장의 문제

현재 임신 확인 방법:
1. 직장검사 (수의사 필수, 수정 후 35일~)
2. 임신진단키트 (혈액/우유, 수정 후 28일~)

→ 전세계 번식 전문 수의사의 **업무 대부분**이 이 작업.
→ 매월 1~2회 새벽 방문, 목장당 2~3시간 소요.

### 2.2 혁신 목표: 센서 데이터만으로 임신 판단

```
수정 후 타임라인:
  0일: 수정 기록
  18~24일: 재발정 윈도우 (미임신 시 발정 재발)
  28~35일: 임신감정 시기 (현재 수의사 방문)
  45일+: 안정기

핵심 가설:
  임신 소: 21일 주기 부근에 활동/반추/체온 변화 없음 (안정)
  미임신 소: 21일 주기 부근에 활동↑ + 반추↓ (재발정)
  → 재발정 유무로 임신/미임신 1차 판별
  → 30~45일 센서 안정성 누적으로 2차 확인
```

### 2.3 현재 구현 상태

v4-pregnancy 엔진:
- stabilityScore (0~1): 재발정 감지, 체온 추세, 활동 패턴, DPI 기반
- 상태: likely_pregnant / uncertain / likely_open
- **한계: 정확도 검증 데이터 부족. 레이블링 축적 필요.**

### 2.4 추가 필요 기능

**2.4.1 발정동기화 프로그램 관리**

수의사가 공태 소에 처방하는 호르몬 프로그램:

| 프로그램 | 프로토콜 | 처치 일정 |
|----------|---------|-----------|
| **PG법** | PG₂α 2회 투여 | Day 0: PG → Day 14: PG → Day 16~17: AI |
| **OVSYNCH** | GnRH-PG-GnRH | Day 0: GnRH → Day 7: PG → Day 9: GnRH → Day 10: AI |
| **G6G** | PG-GnRH-OVSYNCH | Day -8: PG → Day -6: GnRH → Day 0: GnRH → Day 7: PG → Day 9: GnRH → Day 10: AI |
| **Double OVSYNCH** | Presync + OVSYNCH | Day 0: GnRH → Day 7: PG → Day 10: GnRH → Day 17: GnRH → Day 24: PG → Day 26: GnRH → Day 27: AI |

→ 수의사가 프로그램 선택 + 시작일 입력하면, **매일 할 일 리스트**에 "○○번 소 — 오늘 PG₂α 투여" 자동 생성

## 3. 분만 알림

### 3.1 현장의 중요성

- 분만 시 어미소 관찰 → 난산 방지
- 태어난 송아지 **6시간 이내 초유** → 일생의 면역력 결정
- 분만 후 40일 이내 대사성 질병 집중 발생 (유열, 케토시스, 후산정체)

### 3.2 알고리즘 (실측 17,702건 분석)

```
전제: 건유우 OR 임신 후기 (예상 분만일 ± 15일)

분만 예측 타임라인:
  D-14~7: 분만 예정 알림 (달력 기반)
  D-2~1: 체온 하강 시작 (-0.15°C/일)
  D-1~0: 체온 급강하 (-0.32°C, 실측) + 반추 감소(-8.2%) + 활동 불규칙
  H-6~0: 체온 최저점 + 활동 급증 (산통)

분만 감지 조건:
  체온: 개체 7일 평균 대비 -0.3°C 이상 하강
  반추: 5% 이상 감소 동반
  상태: lactationStatus = 'dry' 또는 'dry_off'

심각도:
  분만 6시간 이내 예상 (체온 -0.5°C 이상) → critical
  분만 24시간 이내 예상 (체온 -0.3°C) → warning
```

### 3.3 분만 후 모니터링

분만 확인 후 자동 트리거:
1. 초유 급여 리마인더 (6시간 이내)
2. 분만 후 체온 모니터링 (자궁내막염 조기 감지)
3. DIM 자동 시작 → 케토시스/유열 고위험기 돌입 알림
4. 건유 → 비유 전환 사료 변경 리마인더

## 4. 유방염 조기 발견

### 4.1 현장의 문제

전통적 발견 (이미 늦음):
- 유방 부종, 식욕 저하, 유즙 이상 → **이미 경제적 손실 진행**
- 치료 반응 늦고, 만성화 위험

### 4.2 조기 발견 알고리즘

```
유방염 조기 감지 (임상 증상 전):
  1단계 (subclinical 의심):
    체온: +0.3~0.5°C 상승 (정상 범위 상한)
    반추: 5~10% 감소
    → "유방염 주의 — CMT 검사 권장"

  2단계 (임상형 의심):
    체온: > 39.4°C (실측 평균 39.35°C)
    반추: > 10% 감소
    → "유방염 의심 — 즉시 CMT + 유즙 확인"

  3단계 (급성):
    체온: > 40.0°C
    반추: > 20% 감소 + 활동 감소
    → "급성 유방염 — 수의사 즉시"
```

### 4.3 강화 학습 방향

현재: 체온+반추 기반 (smaXtec 수준)
미래:
- DHI 체세포수(SCC) 데이터 연동 → subclinical 유방염 레이블링
- 치료 결과 피드백 → 유방염 패턴 학습
- 유방별(4분방) 센서 추가 시 분방별 감지

## 5. 대사성 질병 진단

### 5.1 분만 후 40일 — 가장 위험한 시기

| 질병 | 발생 시기 | 센서 패턴 | 현재 구현 |
|------|-----------|-----------|-----------|
| **유열(저칼슘혈증)** | 분만 직후 24~72h | 체온↓ + 활동 급감 + 기립불능 | ❌ 미구현 |
| **케토시스** | DIM 7~70일 | 반추↓(-25%) + 체온 정상 + 활동↓ | ✅ 구현 |
| **후산정체** | 분만 후 12~24h | 체온↑ + 반추↓ + 활동↓ | ❌ 미구현 |
| **자궁내막염** | 분만 후 7~21일 | 체온↑ + 반추↓ + 활동↓ | ⚠️ health_general로 감지 |
| **제4위변위(LDA)** | DIM 14~50일 | 반추 급감(-35%) + 활동↓ | ⚠️ rumination_decrease로 감지 |
| **산독증(SARA)** | 사료 변경 시 | 반추 급감(-30%) + 활동 정상 | ✅ 구현 |
| **기립불능** | 분만 직후 | 활동 0 + 체온 하강 | ❌ 미구현 |

### 5.2 추가 구현 필요

```
유열 감지:
  전제: 분만 후 72시간 이내
  조건: 체온 < 37.5°C + 활동 < 개체 평균의 20% + 반추 급감
  행동: "유열 의심 — 칼슘제 정맥주사 즉시"

후산정체 감지:
  전제: 분만 후 12~48시간
  조건: 체온 > 39.5°C + 반추 감소 + 분만 기록 있음 + 후산 배출 미기록
  행동: "후산정체 의심 — 수의사 확인"

기립불능 감지:
  전제: 분만 후 72시간 이내
  조건: 활동 index < 0.5 (거의 움직임 없음) + 6시간 이상 지속
  행동: "기립불능 — 즉시 응급 처치"
```

## 6. 전염성 질병 — 국가 비상 대응

### 6.1 핵심 가치

단일 목장의 질병 관리가 아니라, **146개 농장 전체의 집단 이상 감지**.
구제역, 럼피스킨, 유행열 — 초기 증상이 모두 **발열 + 반추 저하 + 활동 감소**.
문제: 개체 단위로는 일반 질병과 구분 불가. **집단 패턴**으로만 구분 가능.

### 6.2 집단 질병 감지 알고리즘 (구현 완료)

```
개체 이상 감지 (기존 알람):
  체온 > 39.4°C + 반추 감소 + 활동 감소 → health_general/clinical_condition

집단 이상 감지 (DBSCAN 클러스터링):
  동일 지역(반경 5~50km) 내 2개+ 농장에서 유사 이벤트 동시 발생
  → cluster 생성 → 확산 속도 계산 (farms/day)
  → accelerating이면 epidemic warning

국가 비상 대응 체계:
  1단계 watch: 1개 농장 이상 이벤트
  2단계 warning: 2개+ 농장 클러스터
  3단계 outbreak: 확산 가속 + KAHIS 보고 트리거
```

### 6.3 SEIR 시뮬레이션 (구현 완료)

6개 법정 전염병별 파라미터:
- 구제역(FMD): R0=10, 잠복 5일, 감염 10일
- 럼피스킨(LSD): R0=4, 잠복 7일, 감염 14일
- 유행열(BEF): R0=3, 잠복 5일, 감염 7일
- 브루셀라: R0=2.5, 잠복 30일, 감염 365일
- 결핵: R0=1.8, 잠복 90일
- 탄저: R0=2, 잠복 3일, 치사율 80%

이동제한 시나리오 비교: R0 × 0.4 (60% 감소 효과)
SEIR 피드백 루프: 예측 vs 실제 → R0 자동 보정

## 7. 축산 행정 디지털 전환

### 7.1 CowTalk의 위치

전지전능한 단일 플랫폼이 아니라, **다른 공공 프로그램의 요소(모듈)로 참여 가능한 플랫폼**.

### 7.2 API 기반 참여 가능 영역

| 행정 업무 | CowTalk 제공 API | 활용처 |
|-----------|-----------------|--------|
| 이력제 연동 | 이력번호 → 개체 전체 이력 | 축산물이력시스템 보강 |
| 방역 감시 | 집단 이상 클러스터 API | KAHIS 연동 |
| 등급판정 | 출하 예측 + 유전능력 | 도축장 연동 |
| DHI 연동 | 유량+유질 → 번식+건강 상관 | 젖소검정 고도화 |
| 기상 연동 | THI → 열스트레스 경보 | 축산과학원 연동 |
| 번식 통계 | 수태율/공태일/분만간격 | 농정 통계 |
| 유전 관리 | 근교계수 + 혈통 | 종축 개량 |

### 7.3 멀티테넌시 확장

```
현재: 146농장 단일 인스턴스
```
