## 2. 검증된 표준 수치 (재유도 금지 — 이 값 그대로 사용)

**운영 실측 (대시보드 로그인 기준)**
- 199 농장 · 10,886 두 · 730,491 누적 이벤트 · 센서 가동 99.4%
- 시도별 농장: **충북 77(전국 1위) · 경기 52(수도권 최대·전국 2위) · 충남 27 · 경북 14 · 전북 10 · 전남 8 · 강원 3**
- ⚠️ 경기는 **전국 2위**다. "경기 1위/압도적 1위"는 오류 — 정정 완료. "수도권 최대·전국 2위 기반"으로 표기.

**경기 축산 산업 포지션 (덱 3장 '왜 경기인가')**
- 젖소 사육 **전국 약 41% 점유** (경기, 전국 최대 낙농 거점)
- 수도권 인구 **50%+** 집중 → 최대 소비지도 경기 (생산·소비 동시 중심)
- 전략: **축우(젖소·한우) 중심 디지털 대전환으로 시작 → 양돈·가금 등 타 축종 확산.** "대한민국 낙농은 경기가 중심."

**ROI / 비용·편익 (출처: docs/gyeonggi-briefing-demo-20260625.md §5-A·5-B)**
- 전제: 경기 젖소 100,000두 · 1,500 농장 · 중계기 500만/농가 · 두당 월 15,000원
- 비용: 1년차 255억(중계기 75 + 구독 180) / 매년 180억 / 5년 975억
- 농가 편익: 두당 24만원/년(번식10+질병6+산유5+노동3) → 경기 100,000두 = **240억/년**
- 사회·환경(육성우 36,000두 감축): 사료 432억 + 분뇨 28.8만톤(86억) + 탄소 10.8만톤CO₂eq(108억) = **626억/년**
- 통합 편익 896억(농가240+방역30+사회환경626) vs 비용 180억 → **통합 BCR ≈ 5.0**, 회수 ~2.8년

## 3. 인수위 브리핑 덱 (PPTX)

- **위치**: `docs/gyeonggi-deck/CowTalk_경기인수위_브리핑.pptx` (14장, 발표자 노트 포함, ~15~16분)
- **재생성**: `pip install python-pptx` 후 `python3 docs/gyeonggi-deck/build_deck.py`
  - 출력 경로는 스크립트 하단 `out=` 변수(스크래치패드) — 수정해서 원하는 곳에 저장.
- **구조**: 맥킨지식 — 액션 타이틀(결론부터) + **무엇을→어떻게→그래서 결과는** 흐름(슬라이드 우상단 단계 칩). 거버닝 메시지 = "경기가 디지털 축산행정을 선도한다."
- 14장: 표지 / 제언 / **경기 축산현황(왜 경기·낙농 중심)** / 현황 / 문제 / 해법(4층) / 현황데이터 / 어떻게(알람→행동) / 결과①방역 / 결과②경제 / 결과③사회 / ROI / 로드맵 / 클로징
- 디자인 토큰: 네이비 #0B2239 + 틸 #0E7C7B + 앰버 #C97A1E, 맑은 고딕(eastAsia 지정), 플랫·헤어라인.

- 로고·경기도 CI 삽입, 부록(Q&A·기술 아키텍처) 추가, 톤/문구 미세조정.

## 4. 환경 / 주의사항

- **LibreOffice/soffice 깨짐**: 트리비얼 pptx도 "source file could not be loaded". pptx 렌더 미리보기 불가 → 좌표 검증으로 대체. python-pptx 재오픈은 정상.
- **로컬 풀스택 리허설 데이터**: 경기 농장 52로 시드 정렬됨(`seed-rehearsal.ts`), 등록두수=라이브 정렬. PG는 세션마다 재기동 필요(`pg_ctlcluster 16 main start`, redis, `npm run dev`).
- **git**: 머지 후 `git checkout -B claude/fervent-goldberg-m9d9c2 origin/main`으로 동기화. squash-merge로 원격 브랜치가 stale → 새 커밋 push 시 `--force-with-lease` 필요. GitHub 머지 커밋(noreply@github.com)은 amend 금지(stop hook 오탐).
- **메트릭 계약**: `docs/metrics-contract.md`가 단일 진실원천(D7 두수, D8/D9 currentHeadCount 행정전용, D1 수태율, D3 알림). 두수 관련 수정은 반드시 이 계약 준수.
